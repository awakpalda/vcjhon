Deploy Your Web CV via GitHub Pages

A step-by-step guide to hosting your professional CV online for free

Why GitHub Pages?

GitHub Pages provides free, reliable hosting for static websites directly from your GitHub repository. It's perfect for hosting portfolios, CVs, and personal websites.

1

Create a GitHub Account

If you don't have one, go to github.com and sign up for a free account.

2

Create a New Repository

Click the "+" icon in the top-right and select "New repository". Name it username.github.io (replace "username" with your GitHub username).

3

Upload Your Files

Upload your HTML file (rename it to index.html) to the repository. You can do this via the web interface or using Git commands.

4

Enable GitHub Pages

Go to your repository Settings > Pages. Under "Source", select the branch (usually main) and folder (root). Click "Save".

5

Wait for Deployment

GitHub will build and deploy your site. This usually takes 1-2 minutes. Refresh the page to see the live URL.

6

Access Your CV

Your CV is now live at: https://username.github.io (replace "username" with your GitHub username).

Detailed Instructions

Using the GitHub Web Interface

# Create a new repository named username.github.io
# Click "Add file" > "Upload files"
# Drag and drop your HTML file (named index.html)
# Click "Commit changes"
# Go to Settings > Pages
# Select branch: main, folder: / (root)
# Click "Save"
Pro Tip

Make sure your main HTML file is named index.html - this is the default file GitHub Pages looks for.

Using Git Commands

# Initialize a Git repository
git init

# Add your files
git add .

# Commit your changes
git commit -m "Initial commit"

# Connect to GitHub repository
git remote add origin https://github.com/username/username.github.io.git

# Push to GitHub
git push -u origin main

Custom Domain

You can use a custom domain with GitHub Pages! Just add a CNAME file with your domain name in your repository, and configure DNS settings with your domain registrar.
